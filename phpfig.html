Acerca de PHP FIG

Framework Interoperability Group

www.php-fig.org

La idea detrás del grupo es hablar acerca de las cosas en común que existen entre nuestros proyectos y encontrar una manera para poder trabajar juntos. Nuestra principal audiencia somos nosotros, pero estamos muy al tanto que el resto de la comunidad de PHP lo esta viendo. Si otros amigos quieren adoptar lo que estamos haciendo están bienvenidos para hacerlos, pero ese no es el objetivo.

Los PSR

PHP Standard Recommendation

Por ahora son cinco.

Los propone cualquiera de la comunidad y los aprueban entre todos por votación.

Los PSR

PSR-0 Autoloading Los requisitos obligatorios que deben cumplirse para la interoperabilidad del autoloader.

function autoload($className)
{
    $className = ltrim($className, '\\');
    $fileName  = '';
    $namespace = '';
    if ($lastNsPos = strrpos($className, '\\')) {
        $namespace = substr($className, 0, $lastNsPos);
        $className = substr($className, $lastNsPos + 1);
        $fileName  = str_replace('\\', DIRECTORY_SEPARATOR, $namespace) 
                   . DIRECTORY_SEPARATOR;
    }
    $fileName .= str_replace('_', DIRECTORY_SEPARATOR, $className) . '.php';
    require $fileName;
}

Los PSR

  • Además el archivo tiene que estar en UTF-8 sin BOM
  • Las propiedades pueden tomar cualquier forma pero deben mantener la misma a travéz de todo el código
  • O declaras estructura o ejecutas código

PSR-1 Codificación básica La norma de codificación de los elementos que se requieren para garantizar un alto nivel técnico de interoperabilidad entre el código PHP.

<?php
namespace Pragmore\MiSuperComponente;

class StudlyCaps
{
    const CONSTANTE_EN_MAYUSCULAS = "Y separadas con guión bajo";
    protected $una_propiedad; // Aunque también puede ser $unaPropiedad o $UnaPropiedad
    public function miMetodo()
    {
    }
}
// No puedo escribir aca mismo 
// $sc = new StudlyCaps();

Los PSR

  • 4 espacios sin tabs
  • Salto de línea después de namespace y de los use.
  • Extends e implements en la misma línea pero si hay mas de un implement pueden ir en múltiples líneas.
  • Todas las propiedades con visibilidad.
  • Después de la clase y de un método la llave va abajo.
  • Las estructuras de control tienen que tener un espacio entre los () y los {}.
  • Y más...

PSR-2 Guía de estilo de codificación Extiende el PSR-1.

<?php
namespace Pragmore\MiSuperComponente;

use FooInterfaz;
use BarClase as Bar;

class StudlyCaps extends Bar implements FooInterfaz
{
    public function funcionDeEjemplo($a, $b = null)
    {
        if ($a === $b) {
            bar();
        } elseif ($a > $b) {
            $foo->bar($arg1);
        }
    }

    final public static function bar()
    {
        return true; 
    }
}

Los PSR

PSR-3 Interfaz de Logger El objetivo principal es permitir a todas las librerías usar un objecto Psr\Log\LoggerInterface y escribir logs con él de manera simple y universal.

PSR-4 Autoloader Mejorado Extensión de PSR-0. Básicamente se quita el soporte para las clases que tenían _ (guión bajo) en el mundo pre-namespaces.
